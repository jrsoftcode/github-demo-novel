Git es un sistema de control de versiones distribuido, 
diseñado para rastrear cambios en el código fuente durante el
 desarrollo de software. Fue creado por Linus Torvalds en 
 2005 para ayudar en la gestión del desarrollo del kernel de Linux 
 y se ha convertido en la herramienta de control de versiones más 
 popular y utilizada en la industria del software.

 Características Principales de Git:
Control de Versiones Distribuido:

Git almacena una copia completa del repositorio en cada computadora de 
los desarrolladores que lo clonan. Esto significa que no depende de un 
servidor central para operar, permitiendo trabajar en modo offline y 
colaborativamente sin problemas.
Seguimiento de Cambios:

Git rastrea los cambios que se realizan en los archivos de un proyecto, 
lo que permite a los desarrolladores ver qué modificaciones se han hecho, 
quién las ha realizado y cuándo ocurrieron.
Branching y Merging:

Git facilita la creación de ramas (branches) para probar nuevas características
 o solucionar errores sin afectar el código principal. Las ramas se pueden
  fusionar (merge) posteriormente con la rama principal o con otras ramas.
Colaboración:

Al ser distribuido, Git permite que múltiples desarrolladores trabajen
 simultáneamente en el mismo proyecto, lo cual es esencial para el trabajo en equipo.
Historial y Deshacer Cambios:

Git mantiene un historial completo de todas las modificaciones realizadas
 en el proyecto, lo que permite regresar a versiones anteriores si es necesario.
  También permite revertir cambios no deseados con facilidad.
